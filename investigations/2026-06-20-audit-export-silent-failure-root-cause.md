**PRIMARY root cause (code bug):** `getAuditHistoryToExport` (src/entities/logs.entity.ts:3586) UNCONDITIONALLY `Promise.all([main, archive])`s the audit collection — `AuditHistoryModelArchived.collection.find()` — with NO `checkArchivedDbConnectionRequired` guard (unlike the TAB's `getAuditHistory` and the count/list fns at 1568/1820/2039 which DO guard). The archive Atlas cluster refuses connections from the API node, so with `bufferCommands=false` that find() throws `MongooseError: Cannot call tbl_audit_histories.find() before initial connection is complete`. This runs at controller step 2 (~4597) BEFORE the Chrome/PDF step (~4631), so the export dies here every time. This is exactly why the **tab works** (guarded → main-only for recent data) **but export fails** (unguarded → always hits dead archive). Same unguarded bug in `getReportManageMentDataToExport` (3611). PROVEN: 2 re-triggers (incl. one AFTER the Chrome fix below) → exactly 2 such errors; export still failed → Chrome was never the blocker. `checkArchivedDbConnectionRequired` returns main/archived/both by comparing the date range to now-minus-`KEEP_DATA_ALIVE_IN_CURRENT_MONGODB` days. FIX: add that guard to the export fns + make the archive read resilient (Promise.allSettled / try-catch → archive failure = []).

**GOTCHA — allSettled does NOT catch synchronous throws (cost us a deploy cycle 2026-06-20):** Mongoose throws "Cannot call tbl_audit_histories.find() before initial connection is complete" SYNCHRONOUSLY for a disconnected `bufferCommands=false` collection. The first fix wrote `Promise.allSettled([ main.collection.find().toArray(), archive.collection.find().toArray() ])` — the archive expression is evaluated (and throws) while BUILDING the array, BEFORE allSettled runs, so the sync throw escaped and still killed the export (deployed, re-tested, failed identically). CORRECTION (proven locally): wrap each read in an async fn — `const readCollection = async (m) => m.collection.find(where).sort(sort).toArray();` then `allSettled([readCollection(main), readCollection(archive)])`. An async fn NEVER throws synchronously (any throw → rejected promise), so allSettled captures it. Concurrency preserved. Shipped together with the stuck-flag hardening: `exportAuditHistory` + the report-mgmt export now wrap the `setTimeout` body in try/catch/**finally** (flag reset + socket ack always run, guarded) and the sync catch resets the flag + no longer re-throws (was an unhandled rejection; callers are fire-and-forget). Both pass regression-guard (Go).
