Beta Information


The beta test will run from October 27th, with all devices installed by January 10th, 2022. Post-
beta, all devices will be left in place for ongoing testing and re-used for Pilot testing.


Key goals of the Beta test will be:
  Confirming the installation process and identifying key areas to improve.
  Confirming hardware and software performs as expected.
  Updating installation process for external users in the Pilot.
  Identifying platform areas of improvement.
  Identifying any updates to hub or alarm firmware.
  Confirming if certification is to proceed.



Testing Scope


Exclusions:
  No level of certification testing will be performed — this will be left to Numens/et. al.
  CRM integration will not be tested.
  Quickbooks integration will not be tested.
  No devices will require an electrician to install.

     Use the OpenSignal app to test network strength at installation address.
     Admin (Sam Collins) logs into portal to perform the following:
     Admin adds Property.
     Admin adds Agency.

  Agent installs mobile app.
  Installer installs mobile app.
  All hardware will have 240V leads added, Sam D mentioned he knows some friends who’ll be
  able to do this. TBD details here.



Hardware Installation


Hardware installation is two-part, with the Sensor Hub being installed initially, then alarms.


Sensor Hub
  Installer logs into mobile app.
  Agent assigns Job (on a Property) to an Installer.
  Installer receives push notification about Job.
  Installer takes hardware to location and opens app ready to install.
  Installer scans Sensor Hub QR code with app and installs Sensor Hub hardware, nominally
  behind the fridge.
  Hardware will be installed on 240V leads, so they can be moved around and installed
  anywhere. TBD any additional instructions.
  Installer takes photos of Sensor Hub after it's installed.
  API activates SIM card and polls VERIFY command to Sensor Hub until a response is
  received.
  After 5-10 minutes, the SIM card will be active and the API will receive a VERIFY response
  from the Sensor Hub to indicate it's now online.


Alarms
  After Sensor Hub is installed, Installer will scan the alarm QR code with the app and physically
  install alarm as required.
  Hardware will be installed on 240V leads, so they can be moved around and installed
  anywhere. TBD any additional instructions.
  API will send ADD command to Sensor Hub, which will then link the alarm to that Sensor Hub
  taking around 10 seconds.
  Installer will repeat this process to add additional alarm(s) in property.
  Installer takes photos of alarms after they're installed.
  Installer checks that all alarms have adequate battery charge remaining (TBD above 90%,
  and possibly added via App to show warning).
  After all alarms are installed, Installer will run a Test, where the API will send the Sensor Hub a
  TEST event - and each alarm will sound in sequence.
  With a positive response to the TEST event, installation is done and the Installer taps confirm
  to sign off the Job.




Specific Testing


When testing, please run through Specific Tests and record results, and record any other
feedback not covered in that section under General Feedback. Each of these items will be
elaborated on before Beta commences:



Setup Process


Setup Process — Are the installation instructions easy to follow to install the Sensor Hub and
alarms in a property?
Hardware Installation — Are there any issues during the hardware installation process?



Alarm Hardware


Smoke Alarm — Does the alarm correctly send an alert when detecting smoke?
Humidity — Does the alarm remain silent in the presence of high humidity?
Hush — Does the alarm hush and send a hush notification when pressing the hush button?
Heat — Does the alarm remain silent in a high heat environment?
Sand/dust — Does the alarm remain silent in a sandy/dusty environment?
Tamper test — Does the alarm correctly send a tampered alert when tampered with?
Alarm power loss — Does an AC powered alarm still function correctly when it is unpowered?
General Handling — Can you identify any issues with the general handling of the alarm?



Hub Hardware


Hub power loss — Does the hub still function correctly (and all attached alarms, etc) when it is
unpowered?
