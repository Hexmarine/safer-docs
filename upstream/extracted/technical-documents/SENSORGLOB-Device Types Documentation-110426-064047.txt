    Device Types Documentation

   This documentation provides an overview of the device types used in our system. The
   system interacts with the backend Node.js application and utilizes MongoDB as the
   database. The connection between the system and the backend is secured through a VPN
   (Virtual Private Network). The device types listed below are specifically designed for our
   smoke alarm systems (web), enabling agents, contractors, agencies, and administrators to
   perform various actions related to configuration, invoicing, setups, and more.

Exported Device Types:
1. Android:
    ID: 1
    Code: android
    Name: Android
The Android device type represents devices running the Android operating system. Users can
access the system and perform various actions using Android devices.
2. iOS:
     ID: 2
     Code: ios
     Name: iOS
The iOS device type represents devices running the iOS operating system. Users can access
the system and perform various actions using iOS devices.
3. Web:
     ID: 3
     Code: web
     Name: Web
The Web device type represents devices accessing the system through web browsers. Users
can utilize the web interface to perform various actions and manage the smoke alarm systems.
The system's backend, built on Node.js, provides APIs that facilitate communication between
the frontend and the database. MongoDB serves as the underlying database technology, storing
and retrieving data for the system's functionalities.
The secure connection established via a VPN ensures the confidentiality and integrity of data
exchanged between the system and the backend. This ensures a safe and reliable environment
for performing operations related to smoke alarm system configuration, invoicing, setups, and
more.
